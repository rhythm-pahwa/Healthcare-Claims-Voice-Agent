# Edge Case Test Cases

This document validates the Healthcare Claims Voice Agent's behaviour when unexpected scenarios occur during a conversation. These test cases ensure that the system handles failures securely, provides appropriate guidance to the member, and maintains a consistent conversational experience.

| Test Case ID | Test Case Name                     | Scenario                                                                             | Test Objective                                                                                    | Expected Behaviour                                                                                                                                                                 | Expected Outcome                                                                                                               |
| ------------ | ---------------------------------- | ------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| ET-001       | Invalid Member Authentication      | The member enters an incorrect Member ID during authentication.                      | Verify that the system rejects invalid member credentials without exposing sensitive information. | The authentication request is rejected. The member is informed that the provided details could not be verified. The agent requests the member to enter the details again.          | The authentication process is retried without revealing which credential was incorrect.                                        |
| ET-002       | Authentication Retry Limit Reached | The member enters incorrect authentication details three consecutive times.          | Verify that the system prevents unlimited authentication attempts.                                | Authentication retries stop after the configured limit. The agent informs the member that authentication cannot be completed. The conversation is transferred to Human Escalation. | The conversation is securely handed over to a human representative.                                                            |
| ET-003       | Consent Declined                   | The member declines the consent request.                                             | Verify that the system explains the purpose of consent before terminating the conversation.       | The agent explains why consent is required. The member is asked whether they would like to continue.                                                                               | The member is given another opportunity to provide consent.                                                                    |
| ET-004       | Consent Declined Again             | The member declines consent for a second time.                                       | Verify that the conversation ends when consent is not provided.                                   | The conversation is terminated. No member information is collected. No backend services are invoked.                                                                               | The session ends securely.                                                                                                     |
| ET-005       | Claim Not Found                    | The member provides a Claim ID that does not exist.                                  | Verify that the system handles invalid claim requests gracefully.                                 | The system checks the Claim ID. The member is requested to verify the Claim ID. A second lookup is performed if a corrected Claim ID is provided.                                  | The member either receives the correct claim information or is transferred to Human Escalation if the claim cannot be located. |
| ET-006       | Backend API Timeout                | The backend service does not respond within the expected response time.              | Verify that temporary backend failures are handled correctly.                                     | The system retries the API request. If the retry succeeds, the conversation continues. If the retry fails, the conversation is transferred to Human Escalation.                    | Temporary failures do not immediately interrupt the conversation.                                                              |
| ET-007       | Eligibility Information Not Found  | The member requests eligibility information, but no eligibility record is available. | Verify that the system informs the member when eligibility information cannot be retrieved.       | The member is informed that eligibility information is unavailable. The agent offers additional assistance.                                                                        | The conversation continues without exposing incorrect information.                                                             |
