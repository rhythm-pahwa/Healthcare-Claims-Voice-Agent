# Fallback Test Cases

| Test Case ID | Test Case Name             | Scenario                                                                                                                                                                                       | Expected Behaviour                                                                                                                                                              | Expected Agent Response / Action                                                                                                                       |
| ------------ | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| FT-001       | Unknown Intent             | Member says: “I have an issue with my account.” The request is too generic to identify whether the user needs claim status, eligibility, provider information, benefits, or pre-authorization. | The agent should not assume the user's intent. The agent should ask a clarification question before routing the conversation.                                                   | “I'd be happy to help. Are you calling about your claim, eligibility, benefits, provider information, or pre-authorization?”                           |
| FT-002       | Low Confidence Prediction  | User says: “I want to know if it's approved.” The system cannot determine whether the user is referring to a claim or a pre-authorization request.                                             | The agent should request additional information before routing the conversation to any specific journey.                                                                        | “Could you please let me know whether you're referring to your claim or a pre-authorization request?”                                                  |
| FT-003       | Clarification Successful   | User initially provides an unclear request. After clarification, the user replies: “I'm checking my claim status.”                                                                             | The confidence score should improve. The conversation should continue to the Claims Journey. No human escalation is required.                                                   | Route the user to the Claims Journey and continue with claim status flow.                                                                              |
| FT-004       | Clarification Unsuccessful | Agent asks for clarification. User responds: “I don't know.” The agent asks again, but the user still gives an unclear response.                                                               | The agent should retry according to the configured retry limit. If the intent still cannot be identified, the conversation should be transferred to Human Escalation.           | Ask clarification again. If retry limit is reached, transfer to Human Agent Escalation.                                                                |
| FT-005       | Unsupported Request        | User asks: “Can you book my doctor's appointment?” Appointment scheduling is outside the supported scope of the Healthcare Claims Voice Agent.                                                 | The agent should politely explain the supported services. If the user continues requesting an unsupported service, the conversation should follow the Human Escalation process. | “I can assist you with claim status, eligibility, benefits, provider information, and pre-authorization. Could I help you with one of these services?” |
