# Functional Test Cases

This document validates the core functional flows of the Healthcare Claims Voice Agent. These test cases ensure that the agent can greet the user, collect consent, authenticate the member, route supported requests, complete healthcare journeys, offer additional assistance, and close the conversation successfully.

| Test Case ID | Test Case Name            | Conversation / Scenario                                                                                                                                                                                                                                                                                    | Expected Behaviour                                                                                                                     | Expected Outcome                                                             |
| ------------ | ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| FT-001       | Greeting and Welcome      | **Agent:** Welcome to the Healthcare Claims Voice Assistant. How may I assist you today?<br><br>**User:** Hi.                                                                                                                                                                                              | The agent greets the member. The conversation starts successfully. The agent waits for the user's request.                             | The user is welcomed and the session begins successfully.                    |
| FT-002       | Consent Collection        | **Agent:** Before we continue, I need your consent to access your healthcare information. Do you agree?<br><br>**User:** Yes.                                                                                                                                                                              | The agent records the user's consent. The conversation proceeds to authentication.                                                     | Consent is captured and the user moves to the authentication step.           |

Authentication is successful. A secure session is created.                                           | The member is authenticated and allowed to continue with supported services. |
